==================================================
1. Project Goal
==================================================

Build an interactive carbon study environment that works at global scale and can then be clipped to an area of interest (AOI).

The study environment should help us:

- Fetch open geospatial datasets related to carbon
- Prepare the raw data into aligned layers
- Analyze emission proxies, sequestration proxies, and storage proxies
- Produce maps, tables, and report-style summaries
- Keep the workflow transparent enough for study and reproducibility

Current build strategy:

1. Start with notebooks
2. Move reusable logic into Python modules
3. Wrap the workflow in a lightweight UI later if needed

==================================================
2. Why Global First
==================================================

Global-first is the easier engineering path because:

- Many strong open datasets are already published globally
- The same processing code can be reused for any AOI
- We avoid overfitting the first build to one country
- Australia can still be handled by clipping the global layers

Australia-specific reporting layers can be added after the global base is working.

3. Theory Summary
==================================================

Important principle:
There is no single satellite product that directly gives us "true emissions, sequestration, and storage" at local scale in a complete way.

We need a layered interpretation model.

3.1 Emission layer

Goal:
Represent atmospheric patterns or hotspot signals related to emissions.

Typical products:

- Sentinel-5P methane (CH4)
- Sentinel-5P carbon monoxide (CO)
- Other atmospheric concentration products

Interpretation:

- These products usually show concentration or column abundance
- They are useful for hotspot detection and anomaly analysis
- They are not the same as a verified facility inventory

Use in this project:

- Detect hotspot areas
- Compare periods
- Build anomaly maps

3.2 Sequestration layer

Goal:
Represent carbon uptake or sink behavior over land.

Typical products:

- MiCASA land carbon flux
- Vegetation productivity proxies

Interpretation:

- Often modeled rather than directly measured on the ground
- Useful for regional trends and comparative analysis

Use in this project:

- Estimate whether an AOI is acting more like a sink or a weaker sink over time
- Produce trend summaries

3.3 Storage layer

Goal:
Represent carbon stored in vegetation or soil.

Typical products:

- GEDI biomass
- Soil organic carbon products such as SoilGrids later

Interpretation:

- Storage is a stock, not a flow
- This differs from emissions and sequestration, which are more time-dependent processes

Use in this project:

- Estimate biomass stock context
- Compare land-cover classes
- Support report-ready summaries

3.4 Validation layer

Goal:
Provide context from inventories or official reporting products.

Examples:

- National greenhouse inventory datasets
- Australian DCCEEW data later
- NPI or NGER where relevant

Interpretation:

- Helps keep us honest
- Helps explain whether a satellite or model pattern is plausible

4. v1 Dataset Plan
==================================================

Primary v1 stack:

1. Sentinel-5P CH4
2. MiCASA land carbon flux
3. GEDI biomass
4. ESA WorldCover

Why this stack:

- Global coverage
- Reasonably accessible
- Good balance across emission proxy, sequestration, storage, and land context

Known caution:

- The spatial and temporal resolutions differ
- We must avoid false precision when combining them

5. Standard Workflow Steps
==================================================

Step 1. Define AOI

Accepted inputs:

- GeoJSON polygon
- Country boundary
- State boundary
- Lat/lon plus buffer

Output:

- One AOI geometry in WGS84

Step 2. Define time window

Need:

- Baseline range
- Analysis range

Reason:

- We need a stable reference for anomaly calculations

Step 3. Fetch raw data

For each dataset:

- Connect to source
- Filter by date
- Filter by AOI
- Save metadata on units, resolution, and source

Step 4. Clean and standardize

- Handle missing values
- Clip to AOI
- Normalize date fields
- Preserve units
- Document native resolution

Step 5. Harmonize

- Select an analysis grid
- Resample only when necessary
- Aggregate to a defensible scale

Step 6. Derive variables

Examples:

- Mean AOI methane concentration
- Methane anomaly versus baseline
- Biomass mean and total
- Land carbon flux trend
- Area by land-cover class

Step 7. Generate insights

Expected outputs:

- Hotspot area
- Sequestration trend summary
- Storage estimate summary
- Land-cover contribution summary
- Caveats and confidence notes

Step 8. Export results

Target outputs:

- CSV
- GeoJSON
- Notebook plots
- Markdown summary

8. Rules for Interpretation
==================================================

- Do not claim exact emissions from concentration maps alone
- Do not mix units without explicit conversion notes
- Do not compare rasters at incompatible resolution without aggregation
- Always label whether a layer is measured, modeled, or proxy-based
- Always include caveats in report outputs
